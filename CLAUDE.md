Academic project ‚Äî Deep Learning course, UNH Spring 2026. Team: Varun Gazala, Mohit Raiyani, Jatinkumar Nabhoya.

## Running the Notebooks

Run in order from `notebooks/` (Jupyter, VS Code, or Colab):

1. **`01_phase1_baseline.ipynb`** ‚Äî Creates `images/train|val|test` splits (80/10/10), defines `GymDataset`, runs pretrained COCO YOLOv7 as baseline (mAP ‚âà 0). Run first ‚Äî creates the dataset structure.

2. **`03_augmentation_pipeline.ipynb`** ‚Äî Run before training if you want the augmented dataset. Generates `../dataset_augmented/` with 648 train images (4√ó via copy-paste + albumentations). Val/test are copied unchanged.

3. **`02_phase2_finetuning.ipynb`** ‚Äî Full training + evaluation pipeline (Phases 2 and 3). Switch `RUN_NAME` and `DATASET_DIR` in the config cell to choose dataset:
   ```python
   RUN_NAME    = 'augmented'            # 'original' | 'augmented'
   DATASET_DIR = '../dataset_augmented' # or '../dataset'
   WEIGHTS_OUT = f'../weights/best_{RUN_NAME}.pt'
   ```

Notebooks are CPU-compatible (Mac). GPU will be significantly faster for training.

## Notebook 02 ‚Äî Training Pipeline Detail

`02_phase2_finetuning.ipynb` covers three phases in a single notebook:

**Phase 2A ‚Äî Hyperparameter Search**
- Random search (not Optuna), 8 trials √ó 8 epochs each
- Search space: LR (log-uniform 1e-4 to 1e-2), weight decay (1e-5 to 1e-3), momentum (0.85‚Äì0.95), freeze epochs (2‚Äì6)
- Best HP found: LR=0.00352, momentum=0.936, WD=0.000934, freeze=2

**Phase 2B ‚Äî 2-Phase Fine-Tuning**
- Phase A: freeze backbone, train detection head only (2 epochs warm-up)
- Phase B: unfreeze all layers, full end-to-end (50 epochs max, early stopping patience=15)
- Trained for 42 epochs total (early stop triggered)
- YOLOv7 detection head replaced: 255 channels (80 COCO classes) ‚Üí 30 channels (5 classes)
- 3 output scales √ó 3 anchors, `(nc+5)*3 = 30` output channels

**Phase 3 ‚Äî Final Evaluation**
- Per-class AP, confusion matrix, F1 vs confidence threshold, GT vs prediction visualizations
- TTA: horizontal flip only (4-view multi-scale TTA was deprecated ‚Äî had coordinate bug)
- Results saved to `../results/{RUN_NAME}_metrics.json`

## Augmentation Details (Notebook 03)

Two-stage pipeline applied to training images only:

1. **Copy-Paste**: extracts object crops from training images (246 crops: 75 dumbbell, 91 barbell, 24 kettlebell, 25 resistance_band, 31 pull_up_bar), pastes 2 crops per image with random scale (0.5‚Äì1.2√ó) and max IoU overlap 0.25

2. **Albumentations** (14 transforms): HorizontalFlip, VerticalFlip, Affine, Perspective, PadIfNeeded, RandomCrop, RandomBrightnessContrast, HueSaturationValue, ColorJitter, CLAHE, GaussianBlur, MotionBlur, GaussNoise, CoarseDropout ‚Äî all with `min_visibility=0.30` to drop occluded boxes

Result: 162 ‚Üí 648 training images (4√ó), val/test unchanged.

## What NOT to Change

- `dataset/` folder structure ‚Äî YOLO format requires exactly `images/train`, `labels/train`, etc.
- `data.yaml` class order ‚Äî must match label indices in `.txt` files (dumbbell=0, barbell=1, ...)
- `YOLOV7_DIR` / `DATASET_DIR` variables at the top of each notebook setup cell ‚Äî single source of truth for all path references downstream
- `numpy<2.0.0` pin in `requirements.txt` ‚Äî YOLOv7 internals are incompatible with numpy 2.x

- **203 total images**, 5 custom-collected gym equipment classes

## Common Issues

| Error | Cause | Fix |
|-------|-------|-----|
| `attempt_load` fails | `yolov7.pt` missing from `yolov7/` | Re-run the setup cell ‚Äî it re-downloads (~72 MB) |
| `ModuleNotFoundError: models` | `YOLOV7_DIR` not set or not in `sys.path` | Run cells in order from the top |
| `DATASET_DIR` not found | Running notebook from wrong directory | Must run from `notebooks/` so `../dataset` resolves correctly |
| `weights/best.pt` not found | Training not run yet | Run notebook 02, or copy an existing checkpoint |
| `CoarseDropout` / `GaussNoise` error | albumentations API changed in v1.4 | Notebook 03 auto-detects version and uses correct kwargs |
| numpy 2.x errors in YOLOv7 | numpy version too new | Pin `numpy<2.0.0` ‚Äî already in requirements.txt |
| TTA gives lower mAP than standard | Multi-scale TTA had coordinate bug (fixed) | Use h-flip-only TTA (Phase 3, Section 6 of notebook 02) |
